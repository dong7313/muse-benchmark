Overview

Can LLMs generate CAD that actually fits together?

Existing code-generation benchmarks reward functional correctness (does the program return the right number?) but say nothing about spatial reasoning. MUSE evaluates LLMs on a sharper task: given a natural-language design spec — "a four-legged stool, ~45 cm tall, with cross-stretchers" — can the model write CadQuery code that produces a watertight, manifold, self-intersection-free solid that is also assemblable, functional, and manufacturable?

We curate 106 cases spanning furniture, fixtures, tools, and containers, each paired with a reference STEP file, a 4-view engineering drawing, a 3D render, and a domain-grounded rubric. Candidate code is executed in a sandbox, hard geometric checks are run via OpenCascade, and surviving samples are scored by a vision-language judge (Gemini-3.1-Pro) against the rubric. A sample that fails any upstream check is forced to 0 on every downstream metric — the funnel exposes where each model breaks down.

Evaluation Funnel

Three stages, one fail = downstream zero

Every (case, model, sample) flows through:

Stage 1 · Code Check
Sandbox Success
Does the generated CadQuery script execute without raising? Pure binary signal — no geometry inspection yet.
→ Sandbox
Stage 2 · Geometry Check
Geometric Validity
Four binary OCCT checks — watertight, manifold, self-intersection free, overlap free. Overall means all four passed.
→ Watertight · Manifold · Self-Int · Overlap
Stage 3 · Design Intent
Rubric-Based VLM Judge
Gemini-3.1-Pro scores three pillars against a per-case rubric: Functionality, Manufacturability, Assemblability. Final is the mean.
→ Functionality · Manufacturability · Assemblability
